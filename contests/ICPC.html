Overview
This is the first year that I participated in NAQ alone since my team was busy preparing for 2023 ICPC NENA so we decided to skip this contest to practice on a harder one. Two more teams shown are from the CS392 (Algorithms for Competitive Programming) class that I were working as a course assistance.
Problem Specific Comments
Problem Statement
  • Problem A (6 minutes)
    My first solve of the contest since it is the first problem and it isn't that hard. Pretty much a straightforward problem but the problem statement can throw you of if you're not careful.
  • Problem F (8 minutes)
    Free problem. I solve this problem after A since one of the BU's team have solved it.
  • Problem G (11 minutes)
    A simple find a formula problem.
  • Problem L (16 minutes)
    A simple adhoc problem. You can bruteforce this problem too.
  • Problem H (20 minutes)
    A bruteforce and number theory hybrid. This is the first non-trivial problem since you need to find all divisors of \(n\) in \(O(\sqrt{n})\).
  • Problem D (69 minutes)
    A geometry problem. It is not hard to find the idea of the problem by using linearity of expectation but implementing it is not simple. Took quite a while to find the right libraries to use.
  • Problem E (77 minutes)
    A greedy problem. The input is also nicely sorted.
  • Problem B (94 minutes)
    A string DP problem. Be careful if you are using python to solve this problem since the time limit is quite tight for python.
  • Problem I (171 minutes)
    A surprisingly challenging bruteforce problem because of the sheer number of edge cases in the problem.
  • Problem J (232 minutes)
    My favorite problem in this contest. It is a very beautiful binary search problem once you see it.
  • Problem C (264 minutes)
    A binary search problem with a floating point twist. This problem teaches me about the importance of limiting the amount of iteration when I'm doing binary search on floating points values.
  • Problem K (Upsolve)
    Hard MST + Tree DP problem. Will explain it in the problems section.
