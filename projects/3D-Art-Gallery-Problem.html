3D Art Gallery Problem
3D Art Gallery Problem is an extension of a classic 2D Art Gallery Problem which is an optimization problem on where to place the minimum number of guards inside the 2D shape so that every points in the shape is visible by at least one guard.
In this project, we aim to tackle this problem in 3D space: Given a 3D Mesh, what is the smallest number of guards that we have to place inside the mesh so that every faces of the mesh is visible to at least one guard.
The initial idea is that we should put the guards on the "skeleton" of the mesh. Then, we sample points from the skeleton to be candidate points in which we can put a guard there.
The first step of the project is now split into 2 parts: skeletonize the mesh and handling point visibility.
I mainly work on the visulaization part of the project with polyscope.
To handle the point visibility, the very simple idea is that we can draw a lot of rays out of the guard to see which triangle the ray intersects with.
We have experimented with a few way to sample rays such as, uniformly random, slicing and stacking of a sphere, and utilizing a Fibonacci sphere.
It turns out that the code for point visibility is already existed on gpytoolbox, so we ended up using that in the end.
Next, we have to combine the skeleton with the point visibility to determine where we should put a guard.
First, we sampling points from the skeleton and the repeatedly choose the point that cover the most numbers of unseen triangles until every triangle is covered or no more additional triangle can be covered.
Then, we remove unnecessary guards that no longer see any triangles. (All of their triangle is already covered by other guards.)
In the previous part, it is mentioned that we choose a point until every triangle is covered or no more additional triangle can be covered.
The problem is that our assumption that if we only choose points from the skeleton, we will eventually be able to cover the whole mesh is wrong. Some meshes can't be fully covered using only points on the skeleton.
Since we are running out of time, the quick fix that we decide to go with is just by sampling more points inside the mesh in addition to points from the skeleton.
Despite not being the best solution, this method proved to be better than just using the skeleton by finding the way to place less guards than the former method but taking longer time.
